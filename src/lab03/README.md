# Лабораторная работа №3

## Цель работы

Освоить наследование классов.  
Научиться строить иерархию объектов.  
Закрепить переопределение методов, полиморфизм и переиспользование кода.

## Тема

Интернет-магазин

## Реализованная иерархия классов

В работе используется базовый класс `Product` из предыдущих лабораторных работ.  
На его основе реализованы дочерние классы:

- `DigitalProduct`
- `Service`
- `Accessory`

В файле `base.py` добавлен общий метод `calculate_total()`, который задаёт единый интерфейс для расчёта итоговой стоимости.

## Описание классов

### `Product`
Базовый класс товара.  
Содержит общие поля: название, цену, количество, категорию, скидку и активность товара.  
Поддерживает методы покупки, пополнения склада, деактивации и расчёта цены со скидкой.

### `DigitalProduct`
Цифровой товар.

Дополнительные атрибуты:
- `file_size_mb`
- `license_type`

Дополнительный метод:
- `activate_license()`

Особенность:
- переопределяет `calculate_total()`.

### `Service`
Услуга.

Дополнительные атрибуты:
- `duration_hours`
- `on_site`

Дополнительный метод:
- `book_service()`

Особенность:
- переопределяет `calculate_total()`.

### `Accessory`
Аксессуар.

Дополнительные атрибуты:
- `compatibility`
- `wireless`

Дополнительный метод:
- `connect()`

Особенность:
- использует базовую реализацию `calculate_total()`.

## Что реализовано

В работе реализованы:

- наследование от базового класса;
- три дочерних класса;
- новые атрибуты и методы в дочерних классах;
- использование `super()`;
- переопределение методов;
- полиморфизм через `calculate_total()`;
- работа коллекции с объектами разных типов;
- фильтрация по типу.

## Демонстрация работы

В файле `demo.py` показаны следующие сценарии:

- создание объектов разных типов: `Product`, `DigitalProduct`, `Service`, `Accessory`;
- добавление объектов в общую коллекцию `ProductCatalog`;
- вывод всех объектов;
- вызов методов базового и дочерних классов;
- полиморфизм через общий метод `calculate_total()`;
- проверка типов через `isinstance()`;
- фильтрация объектов по типу;
- использование методов базового класса у наследников.
