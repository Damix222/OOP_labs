## Описание интерфейсов

### 1. Printable
Интерфейс для объектов, которые можно выводить в едином формате.

**Абстрактный метод:**
- `to_string()` — возвращает строковое представление объекта.

### 2. Actionable
Интерфейс для объектов, у которых можно вызвать некоторое действие.

**Абстрактный метод:**
- `do_action()` — выполняет действие, характерное для конкретного класса.

### 3. Sortable
Интерфейс для объектов, которые можно сортировать через единый механизм.

**Абстрактный метод:**
- `sort_key()` — возвращает ключ сортировки.

---

## Реализация в классах

Интерфейсы реализованы в следующих классах:

- `DigitalProduct`
- `Service`
- `Accessory`

Каждый из этих классов реализует сразу несколько интерфейсов:
- `Printable`
- `Actionable`
- `Sortable`

### Отличия поведения

#### DigitalProduct
- `to_string()` выводит данные о цифровом товаре и лицензии
- `do_action()` активирует лицензию
- `sort_key()` сортирует объект как цифровой товар

#### Service
- `to_string()` выводит данные об услуге
- `do_action()` бронирует услугу
- `sort_key()` сортирует объект как услугу

#### Accessory
- `to_string()` выводит данные об аксессуаре
- `do_action()` подключает аксессуар
- `sort_key()` сортирует объект как аксессуар

Таким образом, один и тот же интерфейсный метод в разных классах работает по-разному.
