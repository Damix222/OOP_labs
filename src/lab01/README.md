# Тема
Интернет-магазин

# Выбранная сущность
В этой лабораторной работе реализован класс `Product`, который описывает товар интернет-магазина.

## Описание класса

Класс `Product` содержит:

- название товара;
- цену;
- количество на складе;
- категорию;
- скидку;
- состояние товара.

## Атрибуты класса

- `shop_name` - название магазина;
- `max_discount` - максимальная скидка.

## Закрытые атрибуты экземпляра

- `_name`
- `_price`
- `_stock`
- `_category`
- `_discount`
- `_is_active`

## Что реализовано

В классе реализованы:

- конструктор с проверкой входных данных;
- свойства `@property`;
- `setter` для цены и скидки;
- отдельные методы валидации;
- методы `__str__`, `__repr__`, `__eq__`;
- бизнес-методы:
  - `final_price()`
  - `add_stock()`
  - `buy()`
  - `deactivate()`

## Проверки данных

В программе проверяется:

- название должно быть строкой;
- название не должно быть пустым;
- название должно содержать минимум 2 символа;
- цена должна быть числом;
- цена должна быть больше 0;
- количество должно быть целым числом;
- количество не может быть отрицательным;
- категория должна быть строкой;
- категория не должна быть пустой;
- скидка должна быть числом;
- скидка должна находиться в диапазоне от 0 до `max_discount`.

## Логическое состояние объекта

Товар может быть активным или неактивным.

Реализовано:

- если количество товара равно 0, товар становится неактивным;
- неактивный товар нельзя купить;
- состояние можно изменить методом `deactivate()`;
- после пополнения остатка через `add_stock()` товар снова становится активным.

## Содержимое файла `validate.py`

В отдельный файл `validate.py` вынесены функции:

- `validate_name()`
- `validate_price()`
- `validate_stock()`
- `validate_category()`
- `validate_discount()`

Это позволяет:

- не дублировать код проверок;
- сделать `model.py` более понятным;
- разделить бизнес-логику и валидацию.

## Что показано в `demo.py`

В демонстрационном файле показаны:

- создание объекта;
- вывод объекта;
- сравнение объектов;
- изменение цены и скидки через `setter`;
- доступ к атрибуту класса через класс и экземпляр;
- покупка товара;
- пополнение товара;
- изменение состояния;
- обработка ошибок через `try/except`.
